# 奔奔王国批量兑换

技术栈：

- Node.js 22+
- TypeScript
- sqlite3
- Express

## 功能

- 批量新增账号（按行导入，自动去重）
- 账号列表（搜索、单删、全删）
- 批量兑换（保存 TOKEN、执行兑换、重试失败账号、强制全部设为已兑换）
- 兑换进度实时推送（SSE）
- TapTap 官方公告兑换码自动抓取、入库和自动兑换
- 打开站点需要账号密码登录，账号密码由后端 `.env` 校验
- 支持管理员和临时用户两种角色，临时用户仅可新增账号和查看数据

## 兑换码自动抓取与自动兑换

### 抓取地址

兑换码来源为 TapTap《奔奔王国》官方公告：

```text
https://www.taptap.cn/app/759692/topic?type=official
```

服务端实际通过 TapTap 官方帖子列表接口读取公告内容，并解析官方帖标题和正文里的兑换码。

### 抓取规则

- 服务启动后会立即抓取一次。
- 服务运行期间每 1 分钟抓取一次。
- 抓取到的兑换码会写入 `redeem_codes` 表。
- 同一个兑换码按 `code` 去重，重复抓到只更新来源标题、正文、发布时间和最后发现时间。
- 兑换码发布时间来自 TapTap 帖子的 `publish_time`，没有时使用 `created_time`。
- 只有正文或标题里匹配到 `兑换码：xxx`、`礼包码：xxx`、`CDK：xxx` 这类格式时才会入库。

### 自动兑换规则

- 只有新入库的兑换码才会触发自动兑换。
- 只自动兑换最近 24 小时发布的兑换码；超过 24 小时的历史码只入库，不自动兑换。
- 自动兑换开始后会暂停 TapTap 自动抓取，直到当前自动兑换队列全部执行完毕后再恢复。
- 如果一次抓到多个新兑换码，会按队列顺序逐个兑换，不会并发执行。
- 自动兑换前会检查兑换 TOKEN；如果为空，会自动从兑换站点抓取 TOKEN 并保存。
- 每个兑换码会先写入 `redeem_code_redemptions` 表占位，防止重复触发。
- 自动兑换会对当前所有未删除、未拉黑账号执行兑换。
- 如果自动兑换结束后存在失败账号，会只对失败账号额外重试 1 次；不会继续循环重试。
- 自动兑换结果会记录到 `redeem_code_redemptions` 表，包括状态、总数、成功数、已领取数、失败数和错误信息。

## 启动

```bash
npm install
npm run dev
```

默认地址：`http://localhost:3458`

## 环境变量

复制 `.env.example` 到 `.env` 后可选配置：

- `PORT`：服务端口，默认 `3458`
- `REDEEM_TOKEN`：兑换签名 TOKEN（也可在页面内保存）
- `ADMIN_USERNAME`：登录账号
- `ADMIN_PASSWORD`：登录密码
- `TEMP_USERNAME`：临时账号
- `TEMP_PASSWORD`：临时密码
- `SESSION_SECRET`：登录会话签名密钥
